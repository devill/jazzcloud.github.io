Competencies

Backends

The heart of your backend is business logic. To survive in constantly changing conditions(i.e. real world), your business logic code should be lean.

Ideally the core shouldn't care about persistence, interaction with other services and it should have minimum of dependencies. It should be well-tested and all possible ways to use it should be documented.

Once you achieved it, you can switch storage, payment services provider or any other 3rd party service to another one with ease.

And we want you to have such code. And this is kind of code, we're writing.

Refactoring

There's a huge amount of companies suffering from their current codebase. Busines wants to add new features rapidly, but development can't, because existing code is too complicated and fragile. At this point no innovation is possible anymore, and this is not good.

You already paid a lot to develop your product. Now it is time to invest in architecture, quality and all those things, which you ignored while you main focus was growth.

Our approach

Principles

  • Not to put new dependency until there's more simple way to solve your problem.
  • Follow SOLID smartly
  • Prefer set of micro-apps with single responsibilities over one monolithic app

Simplicity

It is much simpler to build simple software with simple tools! We believe in power of simplicity, and ruby ecosystem provides lots of great simple libraries and tools.

Learn more about simplicity.

Ruby on Rails?

«The main thing you need to know about Rails is when not to use it». We know.

Ruby on Rails has kind of compromised reputation. The complexity of the framework has bad influence on applications written using Rails. We prefer not to use Rails/ActiveRecord as a core of application where it's possible.
